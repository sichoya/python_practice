# 1번: 'Book', 'Library', 'Member'클래스를 정의함.

## 단일 책임 원친(SRP) 사용
- 'Book' 클래스의 경우 책의 정보만을 관리하는 책임을 가진다.

- 'Member' 클래스의 경우 회원 정보와 대출 목록을 관리하는 책임만 가진다.

- 'Library' 클래스의 경우 도서관 전체의 운영 로직을 관리한다.

## 개방-폐쇄 원픽(OCP) 사용
- 'Library' 클래스의 'search_books'메소드에 적용하였다.
+ 기존의 검색 로직을 수정하지 않고, 새로운 검색 기능을 확장 할 수 있습니다.
